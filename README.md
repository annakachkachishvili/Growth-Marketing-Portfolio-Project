# Growth-Marketing-Portfolio-Project

Growth Marketing Portfolio Project
Overview

This repository documents the setup process for the required environment and tools for the portfolio project.

Most of these tools and workflows are not part of my day-to-day work, so my approach was to first fully understand the environment, make sure everything was installed correctly, and avoid rushing into the repository/setup part before verifying that the ecosystem itself was working properly.

The process included:

setting up Cursor;
connecting GitHub;
installing Claude Code and Codex extensions;
testing onboarding and authentication flows;
troubleshooting integration and environment issues along the way.
Tools Installed
Cursor IDE

Installed as the main working environment for the project.

GitHub

Created and connected a GitHub account for repository management and documentation.

Claude Code (Anthropic)

Installed through Cursor Extensions to explore Claude-assisted workflows and onboarding.

Codex (OpenAI)

Installed through Cursor Extensions to test OpenAI-assisted coding workflow integration.

My Process
1. Setting Up the Environment First

Before creating the repository itself, I intentionally focused on setting up the full environment correctly first.

My reasoning was simple:
there was no point rushing into commits and documentation before confirming that the tools, integrations, and onboarding flows were actually functional and understood.

So I first:

installed Cursor;
connected GitHub;
explored the editor environment;
installed the required extensions;
tested onboarding and authentication flows.

Only after that did I move to repository/documentation work.

2. GitHub Integration

GitHub connection initially produced a temporary rate-limit/high-volume error inside Cursor.

Instead of restarting the setup process, I:

verified that the issue was platform-side;
retried the authentication flow;
confirmed the integration status afterward.

The connection completed successfully.

I also encountered an OAuth session expiration message during reconnection, but after checking the integration settings, I confirmed that GitHub had already connected correctly.

3. Extension Setup

I researched and selected the official versions of both extensions rather than installing third-party alternatives.

Claude Code

The extension installed successfully, but authentication required a Claude Pro/Max subscription or API access.

This was useful to discover because installation and actual account connection are treated as separate steps in the onboarding flow.

Codex

The Codex extension installation and onboarding flow felt more directly connected to the existing ChatGPT/OpenAI account ecosystem.

Issues and Problem Solving

One thing I noticed throughout the process is that these tools assume a certain familiarity with IDE workflows and extension systems.

Since this is not my usual environment, I approached everything step-by-step:

verifying results after each stage;
checking whether issues were local or platform-side;
avoiding random reinstall/retry loops before understanding the actual problem.
Final Note

Near the end of the process, my laptop unexpectedly crashed and became unusable, which interrupted the final repository/documentation stage.

Fortunately, by that point I had already completed the environment setup and extension work on my own machine, so the only remaining step was finalizing and publishing the README.

To finish before the deadline, I switched to my husband's MacBook, logged into my accounts there, and completed the repository/documentation submission from that environment instead of restarting the process from scratch.

Honestly, this turned out to be part of the exercise itself:
staying operational, adapting quickly, and finishing the task despite unexpected technical problems.
